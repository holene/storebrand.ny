How to use this

Everything on this page is ready to use. Minor adjustments will come, but as long as you hook onto a specific version of the CSS/JavaScript on Amazon, you will not be affected by them.

Pure form elements are not covered here, but in a separate Web form manual.

The design files can be loaded from a shared repository on Amazon. Use the URL https://elements.storebrand.no/storebrand.ny/version-number/path/filename. There shouldn't be any cross-site scripting issues for pages on *.storebrand.no, but if you encounter some, do tell us. We can fix.

Elements for user interaction should log activity to Google Analytics. You can enable Google Analytics for a clickable element by adding the class "analyze" to the element. The page padding script will add necessary functionality, as long as your markup is in line with the Web manual specification.

Grid and breakpoints

Grid: Based on 12 columns from Bootstrap 3. We can responsively divide the contents horizontally into any combination of 12 in total (12, 6+6, 3+3+3+3, 4+4+4, and so on).

Our screen width breakpoints are:
Extra small = up to 767px, Small = 768-991px, Medium = 992-1199px, Large = 1200px and up.

Typography / Text

Lead paragraph

Lead paragraph (intro): This is bolder. This is where the author grabs the attention of the reader. Lead paragraphs usually include brief answers to all main questions answered by the following text when it comes to who, what, why, when, where, and how.

Code: <p class="intro">...</>
Desktop: font-size: 24px; line-height: 30px; font-weight: 700; margin: 0 0 12px;
Mobile: font-size: 18px; line-height: 24px; font-weight: 700; margin: 0 0 9px;

General paragraph

A paragraph is a passage in a section of text that is about a different subject from the preceding text section, marked by commencing on a new line.

And then the next paragraph follows. It's about something else. The line-height may seem a bit excessive, but it's needed to create sufficient space below characters for the link indicator line, see above.

Code: <p>...</p>
Desktop: font-size: 16px; line-height: 24px; margin: 0 0 9px;
Mobile: font-size: 14px; line-height: 20px; margin: 0 0 6px;

"Small print"

The world of finance in general loves putting important details in small print, but we don't. If a legal text looks really awful, give it the class "disclaimer". We will use that as a sign of something that shouldn't steal too much attention, but must legally be there.

In some cases, like for publishing dates, source references and footnotes, you can use small, like so:
This part of the text is small and not very important, but still possible to read.


Code: <small>Your puny text.<small>

Layout / Spacing

We have a number of classes that can be used on any element to provide some spacing in different ways. Depending on your situation, you may of course want to just use straight CSS styling instead, but these classes come in handy when we want to let our non-IT Web editors use our CMS to work on contents and space things out.

(Note: The actual margins/paddings are NOT 5, 10, ... or 50. That's just a simplification to help people remember the classes. Right now, the numbers are mapped to pixels in the following way: 5 to 6, 10 to 12, 20 to 24, 30 to 30, 40 to 36 and 50 to 48. Sorry about the confusion, if any.)

Vertical spacing

The classes give x pixels (see above) of spacing and are used this way:

class="top-margin-20" => There's a margin-top of 20 (24) pixels added to the element.
class="top-margin-30-mdlg" => A margin-top of 30 (32) pixels is added, but only on md and lg width devices. Devices with xs or sm width gets no margin-top.

The same goes for bottom-margin-10 through bottom-margin-50.

You can use the same pattern for padding, e.g. class="top-padding-50" gives padding-top of 50 (48) to that element, and you can use the -mdlg extension to make it valid only for md and lg width devices.

Spacing on the sides

You can add margins and padding on the sides in the same way, and if desired for large and medium screen widths only, using these classes: left-margin-x and left-margin-x-mdlg, where x can be 5, 10 or 20. And you can of course do the same with right-margin, left-padding and right-padding.

Usage of width

There are also classes for setting the width of an element that otherwise would inherit the full width from its parent element. They were introduced in order to prevent clashes between background graphics pulled to the right when contents pulled to the left needs to be responsive.

There are only two of these classes; class="width-50" and class="width-75", giving widths of 50% and 75%, and you can add the -mdlg extension to let the rule only be valid for md and lg width devices. Devices with sm and xs widths will then keep their original width, usually 100%.

Border use

The class "border-top-and-bottom" adds visible borders at the top and bottom of the element.

The class "border-right" adds a visible border at the right end of the element. It can be used to visually separate contents with a vertical line between different sections/columns.

In some cases you may need to add a common border outside several rows that otherwise exist apart from each other. You can do this by adding a class grayborder with additional side-classes; side124 for top, right and left, side24 for right and left, and side234 for bottom, right and left. Example:

A row with borders on the top (1), right (2) and left (4) side; class="grayborder side124".

A row with borders on the right (2) and left (4) side; class="greyborder side24".

A row with borders on the right (2), bottom (3) and left (4) side; class="greyborder side234".

Tables

By now we should have found ways to present our information without using tables. Still, sometimes we need them. Tables shall never be used to space out contents, but in some cases a table is a good way to present a list.

For a wide table to be responsive, you may have to create two tables with different visibility and different contents for wide and narrow screens, or you can show/hide columns in a wide table.

Tables have no color varieties, they will always follow the style displayed below. Although we can display information for rows as a hover effect, remember that this doesn't work on touch devices.

Our tables have no vertical border markers.

The font-size in tables is 16px by default. You can reduce this to 14px by adding the class "small-text" to the table tag.

The table head always has room for two lines of text, even if there is only one line of text. This is to emphasize where a table ends and begins, which is especially important when two or more tables are close to each other.

Any table can be made so that it is sorted accordingly when a column header is clicked.

Sortable table

When using tables, we typically want to sort them in various ways. We use tablesorter, by adding the class "tablesorter" to the table tag. To make a table sortable, first fill it with information and then sort it like this: $('.tablesorter').tablesorter(); (or use whichever selector that fits your context.)

If you want to sort a column based on a different value than what is actually displayed, you can do this by adding a data-sort-value attribute to the relevant td tags. You must then sort the table using this method: sortTable('#yourSelector');

For more information, please see the original project documentation.

We have used tablesorter in our standard fund lists, see example below.

Standard funds lists

The kind of fund list shown above, but with live data, can easily be added to any page by including a placeholder and calling a script:

Placeholder:

<div id="tableTitle"></div>
<div class="table-responsive">
  <table id="id-for-table-insertion" class="stb-table fund-table tablesorter">
  </table>
</div>

Script:

  listFunds(["storebrand"], "Title above list", "#id-for-table-insertion", "show|hide", "show|hide", "private|professional|whatever");
  setTimeout(function(){sortTable('#list-of-funds');}, 666);

The "listFunds" function takes six parameters:

  1. An array of ISIN or certain predefined strings ("all" for all funds we have data for, "delphi" for all Delphi funds, "storebrand" for all Storebrand funds, "storebrand-private" for private market funds, "storebrand-professional" for professional investors funds, "active" for actively managed funds, "factor" for factor funds, "index" for index funds" and "interest" for interest rate funds).
  2. A title that will be injected above the fund list.
  3. An id of the placeholder table you want to fill with fund data.
  4. A flag to "hide" or "show" a column showing the sustainability rating for the fund.
  5. A flag to "hide" or "show" a column showing the risk (1-7) for the fund.
  6. A flag to "hide" or show links to pages where the fund can be traded, either for "private" or "professional" investors.

You can of course have as many different lists as you want on any single page.

If you're not happy with this presentation, or your application is on a page that is not allowed to retrieve the JSON feed directly, you can always retrieve the raw data instead, and do whatever you want with it.

FAQ

Frequently Asked Questions ("Accordion")

The FAQ "accordion" can be used to initially hide substantial contents on pages that might otherwise seem overwhelming at first glance. (For use only in cases where it's really not possible to reduce the amount of text, of course.) By adding the class "analyze" Google Analytics, tracking of opening of accordion section is enabled.

Example:

Alt om din pensjon

  • Hvordan får jeg oversikt over hva jeg får i pensjon?
    • Har du pensjon i Storebrand, logg inn på dine personlige sider for fullstendig oversikt. Der kan du også hente inn tallene dine fra folketrygden (NAV).
    • Har du fripoliser eller pensjonskapitalbevis fra tidligere arbeidsgivere, finner du disse på minpensjon.no
    • Prøv vår kalkulator som enkelt viser deg hva du kan forvente å få i pensjon. Vi gir deg også en vurdering av hvor mye du bør spare for å få den pensjonen du ønsker deg.
  • Hva består egentlig pensjon av?

    Pensjon består av:

    1. Folketrygden - opptjening betalt av staten. Hvor mye du får, kommer an på hvor gammel du er, ditt lønnsnivå og hvor lenge du har jobbet.
    2. Tjenestepensjon - dette betales av din arbeidsgiver. Her finnes det forskjellige avtaler og det lønner seg å være bevisst på hva du får fra din arbeidsgiver.
    3. AFP - Avtalefestet pensjon. Betales av arbeidsgiver, men fungerer litt som folketrygden. Ikke alle har denne avtalen, så undersøk hos din arbeidsgiver.

Buttons

We differentiate between two types of buttons:

  1. Buttons used for navigation
  2. Buttons used other purposes

For navigation we use the anchor tag markup and for the others we use the standard button tag.

Buttons come in one size only (although with variable width)

Navigation button: Navigate to
Standard button:
Height: 48px
Width: 100-200px
Font: 16px Open Sans Bold

You can enable tracking to Google Analytics of button use by adding the class "analyze" to the a tag that is the button.

The button tag should be used to indicate that you can trig a function on this page. The a tag should be used to indicate that you are now trigging the loading of another page entirely. Visually they will for most users look the same way, ut to a screen reader there is a major difference in how the user will be prepared for what happens when the button is clicked. T

Usage:
<!--For navigation -->
<a class="stb-btn [analyze]">Navigate to</a>

<!--For other purposes -->
<button class="stb-btn [analyze]">Standard</button>
